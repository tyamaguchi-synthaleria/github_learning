## 基本用語

### Git（ギット）
分散型バージョン管理システム。ファイルの変更履歴を記録・管理するためのツール。

### GitHub（ギットハブ）
Gitリポジトリをホスティングし、コラボレーション機能を提供するWebサービスプラットフォーム。

### リポジトリ（Repository）
プロジェクトのファイルと全ての変更履歴を格納する場所。「repo（リポ）」と略されることも多い。

### ローカルリポジトリ（Local Repository）
自分のコンピュータ上に存在するリポジトリ。

### リモートリポジトリ（Remote Repository）
サーバー（GitHubなど）上に存在するリポジトリ。複数人で共有される。

### コミット（Commit）
ファイルの変更をリポジトリに記録すること。変更のスナップショット。各コミットには一意のIDが付与される。

### コミットメッセージ（Commit Message）
コミットに付ける説明文。何を変更したか、なぜ変更したかを記述する。

### コミットハッシュ（Commit Hash）
各コミットに自動的に付与される一意の識別子（SHA-1ハッシュ値）。

## リポジトリ管理

### README（リードミー）
プロジェクトの説明や使い方を記述したファイル。通常はREADME.mdという名前で作成される。

### .gitignore（ギットイグノア）
Gitで管理しないファイルやディレクトリを指定するファイル。ビルド成果物や環境設定ファイルなどを記載する。

### LICENSE（ライセンス）
プロジェクトのライセンス（利用条件）を明記するファイル。MIT、GPL、Apacheなどがある。

### クローン（Clone）
リモートリポジトリを自分のローカル環境にコピーすること。`git clone`コマンドで実行。

### フォーク（Fork）
他人のリポジトリを自分のGitHubアカウントにコピーすること。オリジナルに影響を与えずに変更できる。

### スター（Star）
リポジトリに「お気に入り」や「いいね」をつける機能。プロジェクトへの関心を示す。

### ウォッチ（Watch）
リポジトリの更新通知を受け取る機能。Issueやプルリクエストの動きを追跡できる。

## ブランチとマージ

### ブランチ（Branch）
開発の流れを分岐させる機能。メインのコードに影響を与えずに新機能開発やバグ修正ができる。

### main / master
デフォルトのメインブランチ。製品として公開可能な安定版のコードを保持する。

### HEAD（ヘッド）
現在作業しているブランチの最新コミットを指すポインタ。

### マージ（Merge）
異なるブランチの変更を統合すること。

### マージコンフリクト（Merge Conflict）
マージ時に、同じファイルの同じ箇所が異なる方法で変更されている場合に発生する競合。手動で解決が必要。

### リベース（Rebase）
ブランチの基点を変更する操作。コミット履歴を整理するために使用される。

### ファストフォワード（Fast-forward）
マージ時に、単純にブランチのポインタを進めるだけで済む場合の処理方法。

### チェックアウト（Checkout）
ブランチやコミットを切り替える操作。`git checkout`コマンドで実行。

### スイッチ（Switch）
ブランチを切り替える新しいコマンド。`git switch`（`git checkout`の一部機能を分離）。

## コラボレーション

### プルリクエスト（Pull Request / PR）
自分の変更を他の人にレビューしてもらい、メインブランチにマージしてもらうための依頼。

### コードレビュー（Code Review）
他の人が書いたコードを確認し、フィードバックを提供すること。

### レビュアー（Reviewer）
プルリクエストのコードをレビューする担当者。

### アプルーブ（Approve）
プルリクエストを承認すること。「LGTM (Looks Good To Me)」と表現されることも。

### リクエストチェンジ（Request Changes）
プルリクエストに修正を求めること。

### ドラフトPR（Draft PR）
まだレビュー準備が整っていない作業途中のプルリクエスト。

### コントリビューター（Contributor）
プロジェクトに貢献（コミット、プルリクエスト、Issueなど）した人。

### メンテナー（Maintainer）
プロジェクトの管理者。プルリクエストのマージ権限などを持つ。

### コラボレーター（Collaborator）
リポジトリへの書き込み権限を与えられた共同作業者。

## GitHub特有の機能

### Issue（イシュー）
バグ報告、機能リクエスト、質問などを管理するためのチケットシステム。

### ラベル（Label）
IssueやPRを分類するためのタグ。「bug」「enhancement」「documentation」など。

### マイルストーン（Milestone）
複数のIssueやPRをグループ化し、リリースやスプリントの進捗を管理する機能。

### プロジェクト（Project）
カンバンボード形式でタスクを管理する機能。IssueやPRを整理できる。

### GitHub Actions（ギットハブアクション）
CI/CD（継続的インテグレーション/デリバリー）を実現する自動化ツール。テストやデプロイを自動実行できる。

### ワークフロー（Workflow）
GitHub Actionsで実行される一連の自動化処理の定義。

### GitHub Pages（ギットハブページ）
静的なWebサイトを無料でホスティングできるサービス。

### Gist（ジスト）
コードスニペットやメモを共有するための簡易リポジトリ。

### Wiki（ウィキ）
リポジトリに付属するドキュメント作成機能。プロジェクトのドキュメントを整理できる。

### Release（リリース）
ソフトウェアのバージョンを公開する機能。バイナリファイルの配布にも使える。

### タグ（Tag）
特定のコミットに名前を付けてマークする機能。バージョン番号（v1.0.0など）に使われる。

### Discussions（ディスカッション）
コミュニティとの議論や質問応答のためのフォーラム機能。

### GitHub Copilot（ギットハブコパイロット）
AIを活用したコード補完・生成ツール（有料）。

### Dependabot（ディペンダボット）
依存パッケージの脆弱性やアップデートを自動検出し、PRを作成してくれる機能。

## Git操作

### ステージング（Staging）
コミットする変更を準備エリアに追加すること。`git add`コマンドで実行。

### インデックス（Index）
ステージングエリアの別名。次のコミットに含める変更を一時保存する場所。

### プッシュ（Push）
ローカルリポジトリの変更をリモートリポジトリに送信すること。

### プル（Pull）
リモートリポジトリの変更をローカルリポジトリに取り込むこと。`fetch` + `merge`を同時実行。

### フェッチ（Fetch）
リモートリポジトリの変更情報を取得するが、ローカルブランチには統合しない。

### ディフ（Diff）
ファイルの変更内容を比較して表示すること。

### ログ（Log）
コミット履歴を表示すること。

### スタッシュ（Stash）
作業中の変更を一時的に退避させる機能。ブランチ切り替え時などに使用。

### リセット（Reset）
コミットを取り消す操作。`--soft`、`--mixed`、`--hard`オプションで挙動が異なる。

### リバート（Revert）
特定のコミットの変更を打ち消す新しいコミットを作成する。

### チェリーピック（Cherry-pick）
特定のコミットだけを別のブランチに適用すること。

## ステータスと状態

### ワーキングディレクトリ（Working Directory）
実際に作業しているファイルが存在するディレクトリ。

### アンステージド（Unstaged）
変更されているがまだステージングされていない状態。

### ステージド（Staged）
ステージングされ、次のコミットに含まれる準備ができている状態。

### コミット済み（Committed）
変更がリポジトリに記録された状態。

### トラッキング（Tracking）
Gitが変更を監視しているファイルの状態。

### アントラックド（Untracked）
Gitの管理下にない新しいファイルの状態。

### アップストリーム（Upstream）
ローカルブランチが追跡しているリモートブランチ。

### オリジン（Origin）
クローン元のリモートリポジトリのデフォルト名。

## その他の重要な用語

### SSH（エスエスエイチ）
セキュアシェル。GitHubとの安全な通信に使用される認証方式。

### Personal Access Token（パーソナルアクセストークン）
HTTPSでGitHubにアクセスする際に使用する認証トークン。

### .git ディレクトリ
Gitリポジトリの管理情報が格納される隠しディレクトリ。

### デタッチドHEAD（Detached HEAD）
HEADが特定のブランチではなく、コミットを直接指している状態。

### アップストリームリポジトリ
フォーク元のオリジナルリポジトリのこと。

### マージコミット（Merge Commit）
2つ以上の親コミットを持つ、マージ操作によって作成されるコミット。

### スカッシュ（Squash）
複数のコミットを1つにまとめること。プルリクエストのマージ時によく使われる。

### コンフリクト解決（Conflict Resolution）
マージコンフリクトが発生した際に、どの変更を採用するかを手動で決定する作業。

### CI/CD（シーアイ・シーディー）
Continuous Integration（継続的インテグレーション）とContinuous Delivery/Deployment（継続的デリバリー/デプロイメント）の略。

### ベアリポジトリ（Bare Repository）
作業ディレクトリを持たない、管理情報だけのリポジトリ。サーバー用途で使用。

### サブモジュール（Submodule）
リポジトリの中に別のリポジトリを含める仕組み。

### CODEOWNERS
プルリクエストの自動レビュアー割り当てを定義するファイル。

### Branch Protection（ブランチ保護）
mainブランチなどへの直接コミットを禁止し、プルリクエスト経由でのマージを強制する設定。

## まとめ

これらの用語を理解することで、GitHubでの作業やコミュニケーションがスムーズになります。最初は全てを覚える必要はありません。実際に使いながら、必要な用語から徐々に覚えていきましょう。
